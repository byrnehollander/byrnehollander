23 March 2017

I was surprised by the specificity of Book of the Month’s job posting for the Associate Product Manager position. It identified the future employee’s first project: build the testing infrastructure for the website. I’ve been thinking about this problem for a couple days, considering how I’d approach the problem and guessing at how the site was developed without significant testing.

Armed with Chrome developer tools and the React DevTools extension, I took to the website to see what I could learn. First, I simply tried using all the features on the landing page and seeing if they worked as expected. One thing that caught my eye is how the page responds to minimization: the nav bar becomes a hamburger menu and the images scale (to a limit). Once the images reach a minimum size, the container overflows and the content can only be seen through side scrolling, which isn’t very user friendly (note: this is how it is for all users on the mobile site). Thus, I recommend switching to vertical tiling, as seen below.

I've also made a short gif to demonstrate this problem in action, along with another minor design inconsistency.

After exploring the landing page, I tried making an account so I could see what other features there are and participate in the discussion. While I thought I made an account, I didn't finish checking out, and it was never created. Thus, if the intention is to create an account only for paid subscribers, I'd recommend moving the account password creation step after the checkout. Still, it's possible this is an error. I inspected the call to the server and there was no response (not even one confirming that the chosen email address is not yet registered). Additionally, when I went to a previous step in the check out process, all my progress was lost (this should be a very simple fix).

I returned to the homepage and tried figuring out why a seemingly static site was written in React. I learned that a logged in user has a slightly different view, and I was able to access some of that view by changing a prop. I've made a gif of this process. (note: the react-keydown package can easily improve navigation in modals https://www.npmjs.com/package/react-keydown)
